# Skill 格式自检清单

> 用于检查 `skills/persona-forge.md` 的内容完整性和质量一致性。

---

## 结构完整性

- [ ] 包含 3 步完整工作流：Step 1 问题深析 / Step 2 人设锻造 / Step 3 人设应答
- [ ] 每一步均有明确的输入要求和输出标准
- [ ] 包含核心原则（≥3 条可操作原则）
- [ ] 包含质量守则 / 红线
- [ ] 包含快速命令（≥3 个）

## 模板一致性

- [ ] Skill 中引用的模板路径真实存在
- [ ] 模板中定义的字段结构与 Skill 要求的维度一一对应
- [ ] Skill 中提到的输出格式在模板中有对应位置

## 人设质量

- [ ] 6 个维度（身份/知识/思维/语言/决策/边界）均有明确要求
- [ ] 每个人设维度有"可验证、可追溯"的约束
- [ ] 包含人设自检项（≥4 项）
- [ ] 有"边界诚实"机制（能力边界标注 + 拒绝越界）

## 示例覆盖

- [ ] examples/ 中至少有 2 个不同领域的示例
- [ ] 每个示例完整走完 3 步工作流
- [ ] 示例展示了推理链（人设属性的问题依据）

## 红线与安全

- [ ] 无硬编码凭证 / API Key / 真实个人信息
- [ ] 人设红线在 Skill 中明确列出
- [ ] 应答红线在 Skill 中明确列出

## 一致性

- [ ] README.md 描述的工作流步骤数与 Skill 一致
- [ ] CLAUDE.md 描述的核心 Skill 文件名与 skills/ 目录一致
- [ ] CHANGELOG.md 版本号与 README badge 一致
